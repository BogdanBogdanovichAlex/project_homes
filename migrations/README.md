# Миграции для разделов «Дома и застройщики»

Одноразовые скрипты для подготовки БД на проде. **Запускать строго в этом порядке.**

После каждого запуска удалять файл с сервера, чтобы не оставлять открытый эндпоинт:
```bash
rm /home/bitrix/ext_www/<site>/<script>.php
```

## Порядок запуска

### 1. `migrate_homes_type.php`
Создаёт iblock-тип `homes` («Дома и застройщики»), переносит туда iblock 43 (Готовые дома) и 68 (Застройщики).

**Зависимости:** iblock 43 и 68 уже существуют.
**Идемпотентен:** да — повторный запуск ничего не ломает.

### 2. `migrate_settings.php`
Создаёт iblock «Настройки разделов» (CODE=`homes_settings`) в типе `homes`, добавляет ~13 свойств и 3 элемента (`catalog`, `builders`, `detail`) с дефолтными текстами.

**Зависимости:** Шаг 1 выполнен. **Запомнить ID нового инфоблока** (стейдж: 69) — он используется хелпером `local/php_interface/include/homes_settings.php`. Если на проде ID отличается, поправить хелпер или передавать ID через параметр.

### 3. `fill_settings_props.php`
Дозаполняет свойства 3 элементов настроек (TRUST_LINES, AUDIT_LINES, REVIEWS, FAQ_LINES и т.д.). На stage он понадобился из-за частичного отказа предыдущей миграции — на проде, скорее всего, не нужен. Запускать **только если** какие-то поля остались пустыми.

### 4. `migrate_form_pdf.php`
Добавляет свойство `PDF_PLAN` (тип Файл) в iblock 43. Также пытается добавить поля SCOPE/EMAIL/PROJECT/MESSAGE в форму 7 — на stage не сработал API CFormField::Set, эти поля пришлось создавать руками.

**Зависимости:** iblock 43 существует, форма ID=7 (FORM_CONSULT) существует.

### 5. `fix_form_fields.php`
Меняет тип ответов в форме 7: SCOPE/PROJECT — hidden, MESSAGE — textarea, EMAIL — email. Запускать **после** того, как поля созданы (см. шаг 4 или вручную в админке).

### 6. `migrate_builders.php` _(опционально)_
Создаёт 5 демо-застройщиков в iblock 68 (тех самых, что были fallback-плейсхолдерами в шаблоне). На проде запускать **только если** инфоблок 68 пуст и нужны примеры.

## После всех миграций

1. **Заявить ставки/кнопки калькулятора** через админ-страницу `/bitrix/admin/zemex_homes_calc.php`.
2. **Заявить тексты мастера подбора** через `/bitrix/admin/zemex_homes_wizard.php`.
3. **Загрузить PDF-планировки** для каждого проекта (поле «PDF планировки» в карточке дома).
4. **Создать поля формы 7** в админке вручную, если шаг 4 не справился: Сервисы → Веб-формы → «Получить консультацию» → Вопросы → создать SCOPE / EMAIL / PROJECT / MESSAGE с типами hidden / email / hidden / textarea и сделать их необязательными.
